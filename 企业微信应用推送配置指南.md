## 🔧 配置步骤

### 第一步：创建企业微信应用

#### 1.1 登录企业微信管理后台
- 访问：https://work.weixin.qq.com/
- 使用管理员账号登录

#### 1.2 创建自建应用
1. 点击左侧菜单 **"应用管理"**
2. 点击 **"自建"** 标签
3. 点击 **"创建应用"**
4. 填写应用信息：
   - **应用名称**：`TrendRadar热点推送`
   - **应用logo**：上传一个图标（可选）
   - **应用介绍**：`热点新闻自动推送服务`
5. 点击 **"创建应用"**

#### 1.3 获取应用信息
创建成功后，在应用详情页面获取：
- **AgentId**：类似 `1000002` 的数字
- **Secret**：点击"查看"获取，类似 `abc123xyz...`

**保存这两个值，稍后配置需要用到！**

#### 1.4 设置应用可见范围
1. 在应用详情页面找到 **"可见范围"**
2. 点击 **"添加"**
3. 选择 **"成员"**
4. 添加你自己（或所有需要接收推送的人）
5. 点击 **"确定"**

**重要：如果不设置可见范围，你将收不到消息！**

---

### 第二步：获取企业信息

#### 2.1 获取企业 CorpID
1. 点击左侧菜单 **"我的企业"**
2. 在企业信息页面找到 **"企业ID（CorpID）"**
3. 复制保存（类似 `ww1234567890abcdef`）

**保存这个值，稍后配置需要用到！**

---

### 第三步：获取你的用户 ID

#### 方法一：通过企业微信 App（推荐）
1. 打开企业微信 App
2. 点击右下角 **"我"**
3. 点击头像进入个人信息页面
4. 点击 **"编辑"**
5. 找到 **"账号"** 字段，这就是你的用户 ID
   - 格式类似：`zhangsan` 或 `ZhangSan` 或 `123456`

#### 方法二：通过管理后台
1. 管理后台 → **"通讯录"**
2. 找到你的名字
3. 点击进入详情
4. 查看 **"账号"** 字段

**保存这个值，稍后配置需要用到！**

---

### 第四步：配置 GitHub Secrets

在你的 GitHub 仓库中配置以下 **8 个 Secrets**：

#### 4.1 进入 Secrets 配置页面
1. 打开你的 GitHub 仓库
2. 点击 **"Settings"** 标签
3. 左侧菜单 → **Secrets and variables** → **Actions**

#### 4.2 添加以下 Secrets

| Secret 名称 | 值 | 说明 |
|------------|-----|------|
| `WEWORK_CORP_ID` | `你的企业ID` | 例如：`ww1234567890abcdef` |
| `WEWORK_AGENT_ID` | `你的应用AgentId` | 例如：`1000002` |
| `WEWORK_AGENT_SECRET` | `你的应用Secret` | 例如：`abc123xyz...` |
| `WEWORK_USER_ID` | `你的用户ID` | 例如：`zhangsan` |
| `WEWORK_APP_MSG_TYPE` | `text` | 消息类型（推荐 text） |
| `AI_API_KEY` | `67f3acdf69c12039e2ae68bd7685cdc0:YjMzZjJjOWM0YTcyZWFmMmVmYjY0YjYx` | 讯飞星火 API Key |
| `AI_MODEL` | `openai/astron-code-latest` | AI 模型 |
| `AI_API_BASE` | `https://maas-coding-api.cn-huabei-1.xf-yun.com/v2` | API 地址 |

**注意**：每个 Secret 都要单独添加，不能一次性添加所有！

## 🔧 常见问题排查

### Q1: Actions 运行失败，提示配置不完整？

**检查**：
- GitHub Secrets 是否全部配置（8 个）
- `WEWORK_CORP_ID` 是否正确
- `WEWORK_AGENT_ID` 是否正确
- `WEWORK_AGENT_SECRET` 是否正确
- `WEWORK_USER_ID` 是否正确

### Q2: 运行成功但没有收到消息？

**检查**：
1. **应用可见范围**：是否添加了你的账号
2. **用户 ID**：是否正确（不是姓名，是账号）
3. **企业微信设置**：是否允许接收应用消息
4. **查看 Actions 日志**：是否有错误信息

### Q3: 提示 "获取 access_token 失败"？

**可能原因**：
- `WEWORK_CORP_ID` 错误
- `WEWORK_AGENT_SECRET` 错误
- 网络问题

**解决方法**：
- 检查企业微信管理后台的 CorpID 和 Secret
- 确认 Secret 没有过期

### Q4: 提示 "消息发送失败"？

**可能原因**：
- 用户 ID 不正确
- 用户不在应用可见范围内
- AgentId 错误

**解决方法**：
- 确认用户 ID 是账号，不是姓名
- 在应用详情中确认可见范围包含你的账号
- 检查 AgentId 是否正确

### Q5: AI 分析部分为空？

**检查**：
- `AI_API_KEY` 是否正确配置
- `AI_MODEL` 是否正确
- `AI_API_BASE` 是否正确
- 查看 Actions 日志中的错误信息

### Q6: 提示 "试用期已结束"？

**解决方法**：
- 进入 Actions 页面
- 找到 "Check In" 工作流
- 点击 "Run workflow" 签到续期
- 试用期会重置为 7 天
